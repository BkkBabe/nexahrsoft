NexaHR — Software Development Proposal (draft)


1) Executive summary
Build NexaHR, a cloud-native, all-in-one HRMS that covers time & attendance, payroll, leave & claims, performance appraisal, ATS, project cost tracking, and basic accounting integration. The product will target SMEs initially, with enterprise features and compliance-first security to enable adoption and potential grant funding. 

2) Goals & success metrics
Goals
* Deliver MVP covering core HR functions (attendance, payroll, leave, claims, employee record) within 12 weeks.
* Provide mobile attendance and a web portal for HR and employees.
* Enable payroll export and statutory compliance (local payroll rules) and integration with accounting.
* Achieve enterprise security posture (ISO-grade controls + cloud hosting).
Success metrics
* Time to run full payroll reduced by ≥60% vs manual.
* 80% of pilot users adopt mobile attendance within 30 days.
* Regulatory/compliance export formats supported for target market.
* 99.9% platform availability (SLA target after GA).


3) Target users & personas
* HR Administrator — payroll runs, reports, leave approvals.
* Line Manager — approvals, performance appraisals.
* Employee — punch attendance (mobile), submit leave/claims, view payslips.
* Finance/Accountant — payroll exports, simple accounting links.

4) Scope — MVP vs Phase 2
MVP (12 weeks)
* Employee master data & org structure
* Time & Attendance (web + mobile check-in/out)
* Leave management (apply/approve)
* Payroll engine (gross → net, statutory deductions) with export (CSV/Excel)
* Payslip generation & employee portal
* Basic claims submission/approval
* Role-based access control & SSO readiness
* User level attendance
    * Login and clock attendance through taking photo
    * Send in current location through phone location
Phase 2 (next 3–6 months)
* Performance appraisal module
* Project cost management & timesheet allocation
* Applicant Tracking System (ATS) basics
* Accounting integration (two-way or export + bank reconciliation hooks)
* Advanced reporting & analytics
* Multi-company / multi-entity support


5) Feature mapping (how NexaHR maps to Info-Tech features)
* Mobile Attendance App → Mobile app + geofencing & QR check-in. (Info-tech Systems Ltd)
* Time Attendance Software → Shift scheduling, overtime, roster & exceptions. (Info-tech Systems Ltd)
* Payroll → Payroll engine, statutory formulas, payslips. (Info-tech Systems Ltd)
* Leave Management → Balance tracking, approval workflows. (Info-tech Systems Ltd)
* Claims Management → Expense submission, receipts upload, approval routing. (Info-tech Systems Ltd)
* Performance Appraisal → Review cycles, rating templates (Phase 2). (Info-tech Systems Ltd)
* Project Cost Management → Time allocation to projects, cost reports (Phase 2). (Info-tech Systems Ltd)

6) Architecture & tech stack (recommended)
Architecture
* Cloud-native multi-tenant SaaS hosted in Microsoft Azure (VMs, AKS / App Service, Azure SQL / Postgres). Info-Tech uses Azure; this gives proven disaster recovery and compliance patterns. (Info-tech Systems Ltd)
* API-first microservices for Payroll, Attendance, HRMS, Auth, Reporting.
* Single Sign-On (SSO) / OAuth2 for enterprise customers.
* Mobile apps (iOS/Android) communicating with RESTful APIs.
Tech stack (suggested)
* Backend: Node.js/TypeScript or .NET Core (both Azure-friendly)
* DB: PostgreSQL (primary), Redis (caching)
* Auth: OAuth2 / OpenID Connect, Azure AD B2C optional
* Frontend: React + TypeScript
* Mobile: React Native / Expo
* CI/CD: GitHub Actions / Azure DevOps
* Observability: Prometheus / Grafana / Azure Monitor
* Containerization: Docker + Kubernetes (AKS) or Azure App Service for smaller customers

7) Security & compliance
* Hardened cloud hosting, secure backups, disaster recovery. Info-Tech advertises Azure hosting and disaster recovery—adopt similar patterns. (Info-tech Systems Ltd)
* Apply for certifications over time: ISO 27001, SOC 2, local Data Protection Trustmarks (as Info-Tech has). These improve trust for enterprise customers. (Info-tech Systems Ltd)
* Role based access control, encryption at rest + in transit, audit logging, incident response runbook.

8) Integrations
* Payroll → bank / GIRO or bank file export
* Accounting systems (QuickBooks/Xero/SAP/Local ERP) — start with CSV / flat-file exports, then build direct connectors.
* SSO providers (Azure AD, Okta)
* HRIS data import (CSV/Excel) for onboarding clients.

9) UX / mobile considerations
* Simple employee flows: clock in/out, apply leave, view payslip.
* Manager dashboards for approvals and exceptions.
* Mobile-first attendance: offline capability, photo + geolocation optional.
* Accessibility and multi-language support (optional).
